两道简单的pwn

All_se

  1. 环境: Ubuntu18 pwndbg

  2. 查看保护

  3. 动调

    1. run发现程序输出Hello,world 然后输入
    2. break main在main函数设置断点,ni步过,si步入vulnerable_function,发现有一个read函数,可以构造栈溢出
    3. cyclic 200然后输入,发现返回地址被覆盖为jaab,cyclic -l jaab可以得到缓冲区136字节
    4. 静调可以找到一个getshell函数,得到返回地址0x400320

  4. exp

All_fi

  1. 查看保护

  2. 静调

    1. run发现输出了
      ~~ welcome to ctf ~~
      lets get helloworld for bof
      然后用户输入,cyclic 200输入后发现,读了16个字节

    2. ida打开程序 - 找到main函数,发现函数最后比较了cs:dword_60106C和6E756161h的值,如果相同的话,转入sub_400686(cat flag.txt),即要让cs:dword_60106C等于6E756161h

    3. 程序向unk_601068输入最多16字节的数据,查看内存,发现601068包含4个字节,输入16字节后可以覆盖dword_60106C

    4. 动调验证

      1. 这一题开了保护,无法直接break main

      2. run以后在输入前Ctrl C然后ni步过,输入aaaaaaun('a'*4+p64(0x6e756161))

      3. 这里有比较eax和0x6e756161的值

      4. 查看eax

      5. ni步过

  3. exp
