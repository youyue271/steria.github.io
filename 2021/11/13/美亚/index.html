docker

  • systemctl status docker查看docker是否有启动
  • systemctl start docker启动docker
  • docker images, docker info查看docker版本 docker -v , docker ps , docker ps -a

  • 查看用户账户 cat /etc/passwd

Linux系统取证

Linux易失数据提取

系统信息

  • 系统时间:data -R
  • 系统当前用户:whoc
  • 先起xshell ssh远程连接,查看网络ifconfig,看看网卡有没有起来
  • 查看网卡的配置信息: 重启网卡service network restart, 查看网卡配置信息cat etc/udev/rules.d/70-persistent-net.rules, etc/sysconfig/network-scripts/有网口的配置文件,通过vi修改,如果网卡不识别的话,改成有的网口就行,比如说:eth0不识别,改成eth1,再修改对应的mac地址,根据自己本机的网卡(NAT模式)设置IP地址(网络地址相同,IP地址不同手动配置),再重启网卡
  • uname -a查看linux基本信息
  • last最后的连接记录,
  • /etc/shadow用户信息
  • 网络连接netstat,ipconfig,netstat -lnpt或者-anpt查看tcp的连接端口设置
  • ps -ef列出所有进程(静态),top动态查看进程,M按照内存分配进行排序,P按照CPU占用率进行排序(矿机看程序)
  • 开机自启动服务,chkconfig --list | grep on,cat /etc/rc.d/rc.local,systemctl list-unit-files --type=service | grep enabled
  • cat /proc/cpuinfo查看操作系统版本
  • env查看环境变量
  • 查看用户及组信息
    • w 查看活动用户
    • cut -d: -f1 /etc/passwd 查看系统所有用户
    • cut -d: -f1 /etc/group 查看系统所有组
  • 查看网络,端口信息
    • netstat -an 查看开放端口
    • netstat -lntp 查看所有监听端口
    • netstat -antp 查看已经建立的连接
  • 日志
    • cat /var/log/boot.log 查看Linux系统开机自检过程显示的信息
    • cat /var/log/lastlog记录最后一次用户成功登陆的时间、登陆IP等信
    • cat /var/log/messages记录Linux操作系统常见的系统和服务错误信息
    • cat /var/log/btmp 记录Linux登陆失败的用户、时间以及远程IP地址
    • cat /var/run/utmp 记录有关当前登录的每个用户的信息
    • cat /var/log/wtmp永久记录每个用户登录、注销及系统的启动、停机的事件

Linux非易失数据提取

  • 系统账户 cat /etc/passwd cat /etc/shadow,重点:新增的用户,UID位0的用户,shell环境为/bin/bash的系统服务,空口令站账户

  • 命令行历史:history cat /用户家目录/.bash_history

  • bin目录,命令存储的地方

  • etc配置文件的存放位置

  • var存放日志的地方

  • 防火墙规则 iptables -L

  • 系统日志文件./var/log

    • /var/log/message 系统启动后的信息和错误日志
    • /var/log/secure 与安全相关的日志信息
    • /var/log/maillog与邮件有关的日志信息
    • /var/log/wtmp 永久记录每个用户的登录,注销,以及系统的活动,停机时间,last命令
      • last -x reboot查看重启记录
      • last -x shutdown查看关机记录
      • last -d查看登录记录
    • .bash_history shell日志 tar czvf var_log.tar.gz /var/log
    • Apache日志默认路径:/var/log/httpd/httpd.conf

  • 查找数据库日志find / -name *.cnf

  • mysql默认端口3306

  • rookit程序，用rootkithunter，chkrootkit

  • 打包网站数据 tar cvf my_txt_files.tar find . -type f -name *.jsp

    1. 停止数据库 service mysqld.stop
    2. 备份数据库 mysqldump -h 192.168.1.100 -p 3306 -uroot -ppassword --batabase cmdb > /data/backup/aaadb.sql

  • 日志

    • 应用日志:apache,分析web日志,站点日志,关键词入手select and 1=1等等
    • 如果日志里批量的404里出现了200可能说明网站漏洞扫描出来了
    • 日志格式
      • 事件产生的时间
      • 产生事件的服务器的主机名
      • 产生时间的服务器或程序名
      • 事件的具体信息
